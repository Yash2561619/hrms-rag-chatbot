================================================================================
TECHNOLOGY STACK
================================================================================

Document Processing:
  • pdfplumber - Character-level spatial PDF extraction
  • LangChain - RecursiveCharacterTextSplitter for semantic chunking

Vector Embeddings:
  • ChromaDB - Local vector database (no API, no setup)
  • sentence-transformers - all-MiniLM-L6-v2 (384-dim, 256-token limit)

LLM:
  • Anthropic Claude API - claude-opus-4-5 model

Communication:
  • Flask - Lightweight webhook server
  • Requests - HTTP client for WhatsApp Cloud API
  • WhatsApp Cloud API v25.0 - Message transport

Networking:
  • Cloudflare Tunnel - Secure public URL (free tier available)

  LLM:   Extracts: {leave_type: "Sick Leave", start: "2024-02-15", 
         end: "2024-02-16", reason: "Sick"}
  Bot:   "✓ Leave Application Submitted: Sick Leave from 2024-02-15 to 
         2024-02-16. Your manager will review shortly."

Query 3: Out-of-Scope Question
  User:  "What's the dress code?"
  Context: [Empty - not in HR policy chunks]
  LLM:   Refuses to answer (system prompt prevents inference)
  Bot:   "I don't have information about dress code in the HR policy. 
         Please contact HR: hr@nextai.tech"

================================================================================
ARCHITECTURE HIGHLIGHTS
================================================================================

Cost Efficiency:
  • Full document: 8,000 tokens/query
  • RAG with 3 chunks: 500-800 tokens/query
  • Savings: 90-95% reduction at scale

Speed:
  • Embedding generation: <100ms (CPU)
  • Vector similarity search: <10ms (35-40 chunks)
  • LLM generation: 1-2 seconds
  • Total query latency: ~1-2 seconds

Accuracy:
  • Grounded in actual policy text
  • No hallucination (system prompt enforcement)
  • Handles edge cases and ambiguities
  • Graceful refusal for out-of-domain queries

Scalability:
  • Small corpus: ChromaDB in-memory
  • Large corpus: Migrate to Pinecone, Weaviate
  • High traffic: Add caching and request queuing
  • Corpus updates: Rebuild index (< 1 second)

    response = """Your Benefits:
